# Deploying without Lovable

This project no longer syncs automatically through Lovable. Whenever code
changes touch the database schema, an edge function, or the frontend, the
matching step below needs to be run manually.

## One-time setup

```sh
npm install -g supabase
supabase login
supabase link --project-ref fphpcfecgnfoogfaeihu
```

`supabase login` opens a browser to authenticate with your Supabase account
(the project itself lives on supabase.com, independent of Lovable).

## After pulling changes that add/edit a migration (`supabase/migrations/*.sql`)

```sh
supabase db push
```

If that ever fails or you don't have the CLI set up yet, open the Supabase
dashboard → your project → **SQL Editor** → New query, paste the contents of
the new file(s) in `supabase/migrations/`, in filename order (they're
timestamp-ordered), and run them.

## After pulling changes to an edge function (`supabase/functions/*`)

Deploy just the function(s) that changed:

```sh
supabase functions deploy tag-word
supabase functions deploy pexels-search
supabase functions deploy generate-sentences
```

(`mcp` is auto-generated by the `@lovable.dev/mcp-js` Vite plugin — it's for
exposing your flashcards to MCP clients and isn't required for the app itself
to work, so you can skip deploying it if you don't use that.)

## Secrets edge functions need

Set these once under Supabase dashboard → your project → **Edge Functions**
→ **Manage secrets** (or `supabase secrets set KEY=value`):

- `PEXELS_API_KEY` — used by `pexels-search` for the image lookup in Add Words.
- `LOVABLE_API_KEY` — used by `tag-word` (auto-tagging) and `generate-sentences`.
  This is Lovable's AI gateway key; if it stops working after cancelling your
  Lovable membership, auto-tagging will fail until it's replaced with a
  direct provider key (ask to have `tag-word` migrated off it if that happens).

## Deploying the frontend

```sh
npm install
npm run build
```

`npm run build` produces a static `dist/` folder — deployable to Netlify
(already connected to this GitHub repo, per your installed GitHub Apps),
Vercel, or any static host. No Lovable involvement needed.

## Quick checklist per code change

- [ ] New/changed file in `supabase/migrations/`? → `supabase db push`
- [ ] New/changed file in `supabase/functions/<name>/`? → `supabase functions deploy <name>`
- [ ] Anything else (frontend code)? → picked up automatically by Netlify's
      GitHub-push build, or run `npm run build` yourself if deploying elsewhere.
